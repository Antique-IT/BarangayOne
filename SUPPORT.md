# Support

## Getting Help

If you need help using or contributing to BarangayOne:

- Open a GitHub Issue with a clear title, reproduction steps, and expected vs actual behavior.
- For feature ideas, open an issue describing the use case and desired outcome.

Include the following when reporting technical problems:

- Node.js version
- pnpm version
- Operating system
- Relevant logs or screenshots
- Steps to reproduce

## Troubleshooting Quick Commands

```bash
pnpm install
pnpm run lint
pnpm run build
pnpm prisma db push
pnpm run dev
```

## Security Vulnerabilities

Please do **not** report security vulnerabilities in public issues.

Until a dedicated private security channel is published, contact the maintainers privately through your repository owner/organization communication channel and include:

- A description of the vulnerability
- Steps to reproduce or proof of concept
- Potential impact
- Suggested remediation (if available)

We will acknowledge receipt as soon as possible and coordinate a fix and disclosure timeline.

## Responsible Disclosure Expectations

- Give maintainers reasonable time to investigate and patch.
- Avoid accessing, altering, or destroying data you do not own.
- Avoid service disruption while testing.

## Project Scope Reminder

BarangayOne is community-maintained. Support is best-effort and response times may vary.
